Build Log

What's built, what's synthetic, where it breaks.

Where this stands.

Three honest categories:

Built = architectural, code-enforced.   Synthetic = manually maintained, proves the concept.   Designed = not yet built.

1 layer built. 3 layers proven synthetically. 1 layer designed. The gap between synthetic and architectural is the roadmap.

Layer by layer.

For each layer: what we have (synthetic), what architectural would look like, and where the current version breaks.

Layer 0: Fine-tuned 7B • Built
Voice learned from 4,490 conversation examples. LoRA fine-tuned Qwen 2.5 7B, 3 epochs, loss 3.47→1.55.
What we have
This layer IS architectural. A trained model is architecture by definition — the weights encode the voice.
Architectural
LoRA weights on consumer hardware (RTX 5080). Voice emerges from training, not from prompting. Already there.
Where it breaks
Training data is conversation-only. No structured knowledge. Voice without understanding. The model sounds right but doesn't know right.
Layer 1: ATLAS Retrieval • Synthetic
Living map with 17 section files. Every instance navigates through the map, not by exploring from scratch.
What we have
Manually maintained markdown. Instances read ATLAS.md, follow pointers. Works — but depends on discipline. If an instance skips the map, nothing stops it.
What architectural looks like
Vector + keyword retrieval system. Model queries structured database, not markdown. Retrieval is infrastructure, not a habit.
Where it breaks
Entries go stale. Compressions drift from source. New files get created without map entries. Encoding debt accumulates faster than we can service it.
Layer 2: Typed Routing • Synthetic
REQUIRES / USES / CONTRADICTS / MENTIONS edges in every ATLAS entry since day 1. Defined by propagation consequence.
What we have
Manually typed edge labels in markdown. [REQUIRES] means "breaks if target changes" — but a human types it, and enforcement is a hook that checks, not a graph that enforces.
What architectural looks like
Graph database. Change one node, all [REQUIRES] dependents auto-flag. Propagation is automatic. The graph enforces itself — you can't create an edge without specifying its type.
Where it breaks
Edge types are approximate. An instance writes [USES] when it should be [REQUIRES]. No validation. No automated propagation. The types exist but the consequences are manual.
Layer 3: Persistent Trace • Synthetic
Node files, session logs, MEMORY.md — model reads its own history. Each instance names itself and traces its work.
What we have
Instances write node files, update MEMORY.md, encode learnings. Hooks catch gaps (trace-check, generative-capture). Works — but writing is voluntary, not structural.
What architectural looks like
Automatic trace. Self-modifying inference — the model's state changes based on what it processes. Trace is the processing itself, not a side effect of it.
Where it breaks
Compaction destroys context. New instances get a summary, not the real history. Node files go stale. Some instances encode well, some forget. The trace has gaps.
Layer 4: Attribution Output • Designed
Every generation carries its own proof. The loss function penalizes unexplainable generation.
What we have
Manual citations. "Source: session X, line Y" written by hand. Inconsistent. Some instances cite well, many don't. No enforcement.
What architectural looks like
Self-knowledge loss: loss = predict_right_token() + α × can_you_cite_why(). Unexplainable generation is expensive. Attribution is a training objective.
Where it breaks
It doesn't break yet — it doesn't exist yet. The design is on paper. The training infrastructure exists (Layer 0). The question is whether the loss function actually produces self-knowledge.
Why synthetic matters: Every architectural layer started as a synthetic experiment. "What if we just did this by hand?" — and every one that worked became the spec for what to build in code. Synthetic is not fake. It's the experiment that justifies the architecture.
